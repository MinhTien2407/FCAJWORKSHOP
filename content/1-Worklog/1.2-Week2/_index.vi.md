### Kết quả đạt được tuần 2:


* Hiểu được quy trình di chuyển hệ thống từ môi trường truyền thống lên nền tảng AWS.

* Nắm được các bước đánh giá và lập kế hoạch Migration:
  * Đánh giá hạ tầng hiện có
  * Xác định yêu cầu hệ thống
  * Lựa chọn phương án Migration phù hợp
  * ...

* Thực hành sử dụng AWS VM Import/Export:
  * Import máy chủ ảo lên AWS
  * Export máy chủ ảo
  * Tạo Amazon Machine Image (AMI)
  * Kiểm tra kết quả sau khi Migration
  * ...

* Hiểu quy trình di chuyển máy chủ lên AWS:
  * Chuẩn bị máy chủ nguồn
  * Kiểm tra khả năng tương thích
  * Tối ưu quá trình Migration
  * ...

* Tìm hiểu AWS Database Migration Service (AWS DMS) và cách di chuyển cơ sở dữ liệu lên AWS.

* Thực hành AWS Schema Conversion Tool (AWS SCT):
  * Phân tích cơ sở dữ liệu nguồn
  * Chuyển đổi Database Schema
  * Tạo báo cáo đánh giá Migration
  * ...

* Thực hành Migration cơ sở dữ liệu bằng AWS DMS:
  * Tạo Replication Instance
  * Cấu hình Source và Target Endpoint
  * Tạo Migration Task
  * Theo dõi tiến trình Migration
  * ...

* Hiểu được sự khác nhau giữa Homogeneous Migration và Heterogeneous Migration.

* Kiểm tra và xác thực dữ liệu sau khi Migration:
  * Kiểm tra tính toàn vẹn dữ liệu
  * Kiểm tra kết nối cơ sở dữ liệu
  * Đánh giá khả năng hoạt động của ứng dụng
  * ...

* Nắm được quy trình giảm thiểu rủi ro và xác thực hệ thống sau khi hoàn tất Migration.

* Hoàn thành workshop về AWS Migration:
  * Di chuyển máy chủ bằng AWS VM Import/Export
  * Di chuyển cơ sở dữ liệu bằng AWS DMS và AWS SCT
  * Kiểm tra và xác nhận kết quả Migration
  * ...

* ...
