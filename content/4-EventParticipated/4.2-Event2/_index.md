# Event Report – FCAJ Meetup: Career Development, DevOps, Data Analytics & AWS Architecture

## Purpose of the Event

- Share practical career development experiences in Cloud Computing and DevOps.
- Introduce the real-world responsibilities of DevOps Engineers and Data Analytics Engineers.
- Learn about professional working culture in multinational technology companies.
- Explore AWS best practices for designing scalable cloud architectures.

---

## Speakers

- **Danh Hoàng Hiếu Nghị** – *From First Cloud AI Journey to AWS Partner*
- **Trong H. Trương** – *What Does a DevOps Engineer Really Do?*
- **Cường Nguyễn & Đạt Phạm** – *Real-World Experiences and Corporate Culture*
- **Kiên & Thọ** – *A Scalable URL Shortening Service on AWS*

---

## Key Highlights

### Danh Hoàng Hiếu Nghị – From First Cloud AI Journey to AWS Partner

- Shared his career journey from being an FCAJ participant to becoming an AWS Partner.
- Introduced AWS Student Builder Group and AWS Community Builder programs.
- Explained career opportunities in Cloud Computing and AWS technologies.
- Encouraged students to continuously improve their technical skills and actively participate in technology communities.

### Trong H. Trương – What Does a DevOps Engineer Really Do?

- Explained the daily responsibilities of a DevOps Engineer in modern organizations.
- Highlighted that DevOps is more than CI/CD and Kubernetes—it is a culture focused on automation, collaboration, and system reliability.
- Presented a recommended DevOps learning roadmap:
  - Linux
  - Networking
  - Programming
  - Git & CI/CD
  - Containers
  - Cloud Platforms
- Emphasized the importance of understanding system fundamentals instead of only learning tools.

### Cường Nguyễn & Đạt Phạm – Data Analytics & Corporate Culture

- Shared practical experiences working as Data Analytics Engineers.
- Introduced essential professional skills:
  - Critical Thinking
  - Communication
  - Data Storytelling
  - Problem Solving
- Discussed career progression from intern to experienced engineer.
- Explained recruitment processes and working culture in multinational companies.
- Highlighted the importance of Continuous Improvement and No-Blame Culture in professional environments.

### Kiên & Thọ – A Scalable URL Shortening Service on AWS

- Demonstrated the architecture of a scalable URL shortening application on AWS.
- Explained how to select appropriate AWS services for scalability and high availability.
- Introduced cloud architecture best practices following AWS design principles.
- Showed how multiple AWS services can be integrated to build a reliable production-ready application.

---

## What I Learned

- Gained a clearer understanding of career development in Cloud Computing and AWS.
- Learned the actual responsibilities and required skills of a DevOps Engineer.
- Understood professional working culture and recruitment expectations in global technology companies.
- Improved my understanding of designing scalable cloud architectures using AWS best practices.
- Learned valuable career advice from experienced cloud professionals.

---

## Applying the Knowledge

- Build a structured learning roadmap for Cloud Computing and DevOps.
- Apply AWS architectural best practices when developing cloud applications.
- Improve communication, analytical thinking, and problem-solving skills.
- Design cloud-native applications with scalability and reliability in mind.
- Prepare more effectively for future internships and careers in cloud technologies.

---

## Event Experience

The meetup provided valuable insights from experienced engineers working in cloud computing, DevOps, and data analytics. In addition to gaining technical knowledge, I learned more about career development, professional skills, and workplace culture in modern technology companies.

The speakers emphasized that technical knowledge alone is not enough for long-term success. Building strong communication skills, maintaining a continuous learning mindset, understanding system architecture, and developing problem-solving abilities are equally important for becoming a successful cloud engineer.

> Overall, this event broadened my understanding of career opportunities in Cloud Computing, DevOps, and Data Analytics. The practical experiences shared by the speakers inspired me to continue improving my technical knowledge and professional skills for future opportunities in the cloud industry.
